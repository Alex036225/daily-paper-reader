## 摘要
实时流式联合音视频生成用于角色动画，要求生成器能够说出请求的文本，在各片段间保持视觉一致性，并在严格的播放预算内运行。这些要求很难同时满足：逐块自回归生成会累积文本-音频错位和视觉漂移，而低延迟所需的少步蒸馏常常降低空间多样性和时间质量。我们提出了StreamChar，一个将长时程编排与短窗口音视频去噪分离的流式框架。基于LLM的编排器使用文本和历史上下文生成帧对齐的音频条件，联合音视频DiT在参考帧和运动帧条件下执行局部双向去噪。为了实现高效部署，我们采用两阶段蒸馏流程，首先压缩采样器，然后在在线块推出下微调学生模型。在推出训练期间，一个进度感知指针将部分文本与生成的音频对齐，而汇聚块记忆提供持久的视觉锚定以减少长时程漂移。在短片段和长时程协议上的实验表明，StreamChar在单个H100 GPU上实时运行，并在文本忠实度、音视频同步、视觉质量和流式稳定性方面，与最近的联合和音频驱动基线相比，提供了有利的系统级权衡。

## Abstract
Real-time streaming joint audio-video generation for character animation requires a generator to speak the requested transcript, maintain visual identity across chunks, and run within a strict playback budget. These requirements are difficult to satisfy simultaneously: chunk-wise autoregressive generation can accumulate transcript-audio misalignment and visual drift, while the few-step distillation needed for low latency often degrades spatial diversity and temporal quality. We present StreamChar, a streaming framework that separates long-horizon orchestration from short-window audio-video denoising. An LLM-based orchestrator uses the transcript and historical context to produce frame-aligned audio conditions, and a joint audio-video DiT performs local bidirectional denoising with reference and motion-frame conditioning. For efficient deployment, we use a two-stage distillation pipeline that first compresses the sampler and then fine-tunes the student under online chunk rollouts. A progress-aware pointer aligns partial transcripts with generated audio during rollout training, and a sink-chunk memory provides a persistent visual anchor for reducing long-horizon drift. Experiments on short-clip and long-horizon protocols show that StreamChar runs in real time on a single H100 GPU and provides a favorable system-level trade-off among transcript fidelity, audio-visual synchronization, visual quality, and streaming stability compared with recent joint and audio-driven baselines.

### 1. 核心问题与研究背景
- **研究问题**：实时流式联合生成音视频角色动画面临三大矛盾：长时程文本–语音对齐困难、视觉特征在分块生成中易漂移、低延迟要求的少步蒸馏易导致模式坍塌和多样性下降。
- **问题本质**：单体的多模态扩散模型需要同时承担语义理解、跨块记忆和局部去噪，产生容量竞争与误差累积；蒸馏压缩步数会放大自回归生成中的漂移，两者相互耦合。
- **研究意义**：提出一种解耦架构，将全局规划交给轻量级LLM编排器，将局部高质去噪交给DiT，以实现长时程可观、低延迟、高稳定的流式角色生成。

### 2. 方法论与关键技术
- **整体思路**：StreamChar = LLM编排器（长时程语义规划） + 联合音视频DiT（短窗口双向去噪），通过两阶段蒸馏实现实时推理。
- **LLM 编排器**：
  - 接收文本、参考/历史音频特征，生成帧对齐的连续音频条件 `ca`。
  - 不生成离散音频token，而是通过端到端训练联合DiT输出条件，避免自回归累积错误。
  - 编排器使用 Qwen2.5‑3B 初始化，与 DiT 联合预训练。
- **联合音视频 DiT**：
  - 对噪声视频和音频潜变量进行双向自注意力去噪，同时添加参考帧和运动帧条件。
  - 采用模态感知的混合专家前馈网络（MoE‑FFN），视频和音频token共享注意力但路由到不同专家。
  - 音频对齐通过轻量Transformer编码器将 `ca` 融合到音频潜变量，语音与画面在同一时间尺度上通过RoPE频率缩放（音频RoPE基频为视频的1/4）对齐。
  - 使用**时间步不变条件**和**非对称注意力掩码**：参考/运动帧保持干净潜变量（t=0），且不与噪声token交互，支持预计算以加速推理。
- **流式推理与蒸馏**：
  - **块内双向注意力**：每个生成块内使用全双向注意力，以保证质量；块间通过运动帧（先前生成块）传递连续性。
  - **两阶段蒸馏**：
    - **阶段一（步数压缩）**：使用分布匹配蒸馏（DMD）将50步教师压缩为4步学生，仅针对单块生成。
    - **阶段二（在线推演微调）**：学生自回归生成多个连续块，使用自身输出作为运动帧来计算DMD损失，同时借助进度感知指针（PAP）和汇聚块记忆（sink‑chunk）稳定训练。
  - **进度感知指针（PAP）**：预测每个生成块对应的文本结束索引，用于截断文本，保证自回归推演中损失计算与音频进度对齐。
  - **汇聚块记忆（sink‑chunk）**：每个生成序列的第一块被持续用作后续块的远程条件，抑制长视频中的视觉漂移和模态坍塌。
- **训练流程**：
  - 数据：语音数据（Emilia）、音视频数据（SpeakerVid、TalkVid、OpenHumanVid）分阶段预训练。
  - 预训练后，执行蒸馏阶段一（600步）、阶段二（400步），学习率分别为2e‑6和4e‑7。

### 3. 实验设计
- **数据集**：短片段测试使用 EMTD 数据集（150个10秒样本）；长时程测试使用 EMTD 长时程子集（50个样本，文本>300词，生成5分钟流）。
- **Benchmark 指标**：
  - 音视频同步：Sync‑C、Sync‑D。
  - 视觉质量：FID、FVD、VBench‑2.0 的人体解剖与身份得分。
  - 语音可懂度与文本对齐：WER。
  - 长时程漂移：每30秒计算片末5秒与片初5秒的绝对质量差，取最大值（Quality Drift），并评估 Dynamic 得分（运动多样性）。

  - 非流式联合音视频生成模型（LTX‑2、OVI、MagiHuman）——作为质量参考。
- **用户研究**：24名参与者、50个随机案例，使用 GSB（好‑同‑差）法比较运动自然性、唇形同步和运动丰富度。

### 4. 资源与算力

### 6. 主要结论与发现
- 通过解耦编排器与 DiT，StreamChar 在保持实时（约1.34 s/块）的同时，改善了文本–语音对齐（WER 3.5‑3.6%），竞争中接近非流式大模型而不牺牲稳定性。
- 两阶段蒸馏 + 汇聚块机制能有效抑制长时程漂移，Drift 从 0.0304 降至 0.0067，同时保持最高运动多样性（Dynamic=1.0）。
- 在视觉质量和同步性上优于或可比肩专用音频驱动模型，尤其在人体解剖评分和FID上表现出色。
- 用户研究显示该方法在运动自然性和同步上优于流式基线。

### 7. 优点
- **架构解耦**：将全局语义规划从扩散去噪分离，缓解单一模型的容量冲突，使长文本驱动更稳定。
- **两阶段蒸馏解耦**：先压缩步数再训练自回归一致性，减少梯度冲突，防止模态坍塌。
- **汇聚块记忆**：以极小代价有效缓解长时程视觉漂移，同时保持运动多样性。
- **轻量高效**：基于5B视频骨干和3B LLM，实现单H100实时推理，优于多家14B+的参数规模。
- **全面评估**：同时涵盖同步、质量、智能度和长期稳定性，对比基线合理且透明。

### 8. 不足与局限
- 音频驱动基线并非完整的文生音视频模型，仅用生成音频作为输入，因此对语音生成能力的比较不直接。
- 实时性能仅在一张 H100 上验证，低频端或消费级设备可能需要进一步的优化。
- 训练数据中的视频/文本最长仅20秒，虽通过历史截断保持一致，但未验证超长（>数小时）场景的稳健性。
- 蒸馏损失依赖ASR标签，PAP的准确性在极端口音或噪声条件下可能影响对齐训练。
- 长时程漂移指标仅依靠VBench质量差值，未包含更细粒度的身份保持或动作一致性评估。
