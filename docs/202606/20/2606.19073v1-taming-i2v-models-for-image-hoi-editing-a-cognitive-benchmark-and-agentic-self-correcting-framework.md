## 摘要
当前的图像编辑方法擅长处理静态属性，但在复杂的人-物交互（HOI）方面表现不佳，这一关键挑战尚未被现有基准所解决，因为它们将HOI与静态属性混为一谈，依赖全局指标无法同时评估动态交互的有效性和纠缠的人-物对保留。因此，我们首先引入了HOI-Edit，一个具有三个渐进认知层次的综合基准，其特点是自动化指标HOI-Eval，通过让视觉语言模型（VLM）对包含接地人-物对的图像进行思考后问答，可靠地评估实例级交互。考虑到该任务重塑动态关系的本质，我们对图像到视频（I2V）模型进行了基准测试，发现它们因其时序生成能力而天然适合动态编辑。至关重要的是，除了优越的性能外，这种能力还提供了“失败过程的重放”，为错误原因提供了独特的可诊断性。因此，我们提出了SCPE（自我纠正过程编辑），一种新颖的智能体自我纠正框架，通过迭代优化的提示约束I2V模型的生成，使生成的视频能更准确地呈现目标HOI。从这些视频中提取的帧即为最终编辑结果。在HOI-Edit上，SCPE在交互方面取得了与最先进的编辑模型（如Nano Banana）相竞争的性能。代码见https://github.com/oceanflowlab/HOI-Edit。

## Abstract
Current image editing methods excel at static attributes but fail at complex Human-Object Interactions (HOI), a critical challenge unaddressed by existing benchmarks that conflate HOI with static attributes, relying on global metrics incapable of simultaneously assessing dynamic interaction validity and entangled human-object pair preservation. Thus, we first introduce HOI-Edit, a comprehensive benchmark with three progressive cognitive levels, which features an automated metric HOI-Eval that reliably evaluates instance-level interaction by letting VLM Q&A after thinking with images containing grounded Human-Object pairs. Considering the task's essence of remodeling dynamic relationships, we benchmark Image-to-Video (I2V) models, finding them inherently suited for dynamic editing due to their temporal generation capabilities. Crucially, beyond superior performance, this capability provides a "replay of the failure process," offering unique diagnosability into why errors occur. We thus propose SCPE (Self-Correcting Process Editing), a novel, agentic self-correcting framework that constrains the generation of I2V models through iteratively refined prompts, enabling the generated videos to more accurately present the target HOI. Extracted frames from these videos are the final editing results. On HOI-Edit, SCPE achieves performance competitive with state-of-the-art (SOTA) editing models like Nano Banana on interaction. Code is available at https://github.com/oceanflowlab/HOI-Edit.

## 1. 核心问题与整体含义
- 当前图像编辑方法在修改静态属性（如颜色、纹理）方面表现出色，但在 **人-物交互（Human-Object Interaction, HOI）编辑** 上存在根本性缺陷。HOI 编辑要求模型在保持原有主客体身份的前提下，生成符合语境、逻辑合理的动态交互过程（如“拿起”“骑行”“切割”），而非简单替换像素。
- 现有基准通常将 HOI 与静态属性混为一谈，依赖全局相似度指标（如 CLIP-score），无法同时评估**动态交互的有效性**与**纠缠的人-物对身份保留**。
- 本工作的核心目标是：（1）建立一个**分层次认知的 HOI 编辑基准**，推动模型从静态修图转向动态过程模拟；（2）探索**图像到视频（I2V）模型在动态交互编辑上的潜力**，并利用视频生成的“失败过程重放”特性实现智能体自我纠正。

## 2. 方法论
### 2.1 HOI-Edit 基准与 HOI-Eval 评估指标
- **HOI-Edit 数据集**：包含三个递进认知层级：
  - **L1 基础编辑**：创建、删除、修改交互，同时保持人与物的身份。
  - **L2 上下文空间理解**：根据空间描述选择目标（如“最上面的苹果”）并控制最终位置（如“把花放进花瓶”）。
  - **L3 因果与物理推理**：推导未明说的前提步骤（如“开盖”后才能“搅拌”）以及物理效应（如光照变化、非刚性形变）。
- **HOI-Eval 自动评估**：基于“使用配对区域进行思考”范式。
  1. **实体定位与关联**：将原图中人工标注的主客体边界框通过跟踪（SAM 2）传播到编辑后图像，确保区域对应。
  2. **身份一致性验证**：裁剪并拼接原图与编辑图的对应区域，让 VLM 判断相似度，得到 \(H\)（人）和 \(O\)（物）分数。
  3. **交互与合理性评估**：询问交互是否发生，得到交互分 \(I\)；通过上下文感知问题（如“镊子是否被使用”）验证空间、逻辑、物理约束；引入 **I+Q&A** 指标，仅当上下文问题回答正确时保留交互分，否则置零，以此惩罚不合理编辑。

### 2.2 SCPE（Self-Correcting Process Editing）框架
- **核心思想**：利用 I2V 模型生成视频的过程信息进行错误诊断，通过**可进化的 Playbook（知识手册）** 和**多智能体闭环**迭代优化提示词，从而纠正交互错误。
- **系统组件**：
  - **Generator（生成器）**：根据输入图像、简单指令和当前 Playbook 生成增强后的详细视频提示。
  - **Analyzer（分析师）**：观看生成的视频，评估交互是否成功，输出失败分析报告。
  - **Reflector（反思者）**：将具体失败案例抽象为通用教训（如“模型倾向选择最近物体，需强否定词聚焦目标”）。
  - **Curator（策展者）**：将通用教训增量更新至 Playbook，避免冗余。Playbook 包含策略、陷阱和可重用模板。
- **流程**：初始指令 → Playbook 增强提示 → I2V 生成视频 → Analyzer 诊断 → Reflector 提炼见解 → Curator 更新 Playbook → 下一次生成循环。最终从生成的视频中通过**语义帧选择器**（两阶段：生成完成问题→寻找最早满足条件的清晰帧）选出最优一帧作为编辑结果。
- **关键特点**：梯度自由，可适配不同视频生成骨干；利用视频过程提供“可诊断性”，将黑箱错误转化为可纠正的逻辑过程。

## 6. 主要结论与发现
- **视频时间先验至关重要**：I2V 模型天然适合 HOI 编辑，其生成过程能暴露失败原因，而静态模型仅呈现最终错误结果。
- **SCPE 显著提升交互编辑能力**：在 HOI-Edit 所有认知层级上，SCPE 使 Wan 2.2 的交互分全面超过商业 SOTA Nano Banana，并在开源模型中达到最优。
- **Playbook 实现经验传递**：基于失败案例的图像反馈远比盲目的提示重写有效；Playbook 存储的通用规则能跨样本纠正常见错误模式。
- **框架可扩展**：SCPE 梯度自由，可无缝适配更快速或更强的视频生成主干，且不依赖特定 VLM。

## 7. 优点
- **定义了层次化 HOI 编辑基准**：首次将任务拆解为基础交互、空间理解、因果物理推理，提供细粒度评估工具。
- **创新的评估范式**：HOI-Eval 通过区域接地和任务特化问题，精确衡量身份保留与交互合理性，与人眼判断高度相关。
- **巧妙的错误利用**：将 I2V 的“失败过程”转化为可诊断信息，指导自我纠正，突破了传统图像模型的黑箱限制。
- **实用的智能体设计**：Playbook 增量更新、多角色协同、梯度无关，使其易于部署、成本低。
- **实验扎实**：多角度验证有效性，排除评估器偏差，展示框架的鲁棒性与泛化能力。

## 8. 不足与局限
- **推理延迟**：尽管可通过加速模型缓解，当前范式仍比静态编辑慢；完整迭代需要额外推理时间。
- **背景稳定性**：依赖 I2V 模型，可能引入相机运动、背景飘移或非目标物体位移，影响场景保真度（尽管已有一定缓解）。
- **数据规模有限**：HOI-Edit 基准目前共 705 个样本，交互类别和场景多样性尚待扩充。
- **物理模拟能力边界**：SCPE 仍通过提示词约束生成，而非真实物理引擎，复杂物理效应的准确性受限于底层生成模型能力。
- **身份一致性权衡**：有时为完成大幅空间重定位，I2V 生成可能导致主体外观轻微变形，导致身份分数略低于专门优化静态保留的商业模型。
