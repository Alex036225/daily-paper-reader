## 摘要
我们推出 Wan-Streamer，一个原生流式、端到端的交互基础模型，专为实时、低延迟、全双工视听交互从头设计。Wan-Streamer 在单个 Transformer 中无缝地将语言、音频和视频同时作为输入和输出进行建模，序列表示为交错的视觉、音频和文本输入令牌与视觉、音频和文本输出令牌，通过块因果注意力协调以实现增量流式传输。与依赖独立 VAD、ASR、语言、TTS、音频驱动动画或视频生成模块的级联交互系统不同，Wan-Streamer 不依赖于外部语言、语音、化身或视频生成模块：感知、推理、生成、响应时机、回合管理和跨模态同步在一个统一模型中联合学习，减少了流水线延迟和误差累积。为了支持自然的视听响应性，我们围绕流式传输能力重新设计了整个技术栈，包括因果编码器、因果解码器、块因果注意力和低延迟多模态令牌调度，使得在 25 fps 下流式单元短至 160 毫秒。Wan-Streamer 实现了约 200 毫秒模型端响应延迟，当加上 350 毫秒的双向网络延迟时，总交互延迟约为 550 毫秒，支持亚秒级双工视听通信。这些结果使 Wan-Streamer 成为一个用于低延迟流式交互的统一、端到端、多模态交互基础模型。

## Abstract
We present Wan-Streamer, a native-streaming, end-to-end interactive foundation model designed from the ground up for real-time, low-latency, full-duplex audio-visual interaction. Wan-Streamer seamlessly models language, audio, and video as both input and output within a single Transformer, where the sequence is represented as interleaved visual, audio, and text input tokens together with visual, audio, and text output tokens, coordinated by block-causal attention for incremental streaming. Unlike cascaded interactive systems that rely on separate VAD, ASR, language, TTS, audio-driven animation, or video-generation modules, Wan-Streamer does not rely on external language, speech, avatar, or video-generation modules: perception, reasoning, generation, response timing, turn management, and cross-modal synchronization are learned jointly within one unified model, reducing pipeline latency and error accumulation. To support natural audio-visual responsiveness, we redesign the entire stack around streamability, including causal encoders, causal decoders, block-causal attention, and low-latency multimodal token scheduling, enabling streaming units as short as 160 ms at 25 fps. Wan-Streamer achieves approximately 200 ms model-side response latency and approximately 550 ms total interaction latency when combined with 350 ms bidirectional network latency, supporting sub-second duplex audio-visual communication. These results position Wan-Streamer as a unified, end-to-end, multimodal interactive foundation model for low-latency streaming interaction.

### 1. 论文的核心问题与整体含义
当前实时视听交互系统普遍采用“级联式”（cascaded）架构：由独立的 VAD（语音活动检测）、ASR（自动语音识别）、语言模型、TTS（语音合成）与音频驱动动画 / 视频生成模块串联而成。这种流水线在工作时会产生模块边界等待，引发延迟与误差的逐级累积。与此同时，真实的类人交互是**流式的、全双工的**（full-duplex）：人在听的同时也在看，在说的同时也在接收反馈，感知与表达在时间上高度重叠。然而，大多数现有系统要么只输出语音 / 文本，要么将视觉生成作为后置模块，难以在同一个因果流中同步完成感知、推理、语音生成、面部动作生成以及响应时机的协同优化。  
**整体含义**：Wan‑Streamer 是**第一个**从设计之初就以“原生流式、全双工、端到端”为目标的多模态交互基础模型，力图用单一 Transformer 将所有模态的输入 / 输出统一为一个因果序列，消除对外部独立模块的依赖，从而将响应延迟降低到亚秒级，并从机理上保证视听同步与长期交互一致性。

### 2. 论文提出的方法论

#### 2.1 核心思想：全模态因果流
- 将一次交互视为一个连续的因果流，序列由 **交错的视觉、音频、文本输入令牌** 与 **视觉、音频、文本输出令牌** 组成。
- 用户输入 \(u_k\) （文本 / 音频 / 视频）与代理响应 \(y_k\) （文本 / 音频 / 视频）基于**块因果注意力（block‑causal attention）** 实现逐流式单元（160 ms, 25 fps）的增量生成，整个历史均可用于当前步的条件化。

#### 2.2 关键技术细节

- **完全因果的 VAE 与编解码器**  
  为支持流式，音频和视频采用**严格因果**的变分自编码器（VAE），仅使用过去信息进行潜在编码与解码；同时视频编码 / 解码均设计为因果结构。
  
- **联合训练范式**  
  文本部分使用离散 token 的下一 token 预测（交叉熵损失）；音频与视频部分则使用连续潜在变量，通过 **conditional flow matching** 联合生成，采用公式  
  \[
  z_m^\tau = (1-\tau) z_m^0 + \tau \epsilon_m,
  \]
  并学习速度场 \(f_\theta(z_a^\tau, z_v^\tau, c_k, \tau)\) 以同时去噪音频与视频潜在表示，使唇形、表情、韵律在生成前即耦合。

- **三阶段训练**  
  1. **独立任务预训练**：从语言模型初始化，混合图像 / 音频 / 视频理解、对话、图像 / 音频 / 视频生成等任务，让感知与生成能力在同一序列模型中初对齐。  
  2. **端到端交互训练**：使用真实双工交互数据，模型观察用户在说话时的打断、重叠、停顿等行为，学习同步的“听态反馈”（nod、gaze 等）和主动说话。  
  3. **蒸馏与自推演（self‑forcing）**：用带 CFG 的强教师模型蒸馏学生模型，吸收 CFG 效果、减少求解步数；并通过滚动自推演与分布匹配（distribution matching）缓解训练‑推理不一致，提升长序列生成质量。

- **推理：Thinker‑Performer 分离流水线**  
  虽为单一模型，部署时拆分为 Thinker（含因果编码器、语言预测、KV‑cache 构造、因果解码器）和 Performer（仅负责 flow‑matching 潜在生成）。  
  每流式步 k：  
  - Thinker 编码当前用户观察，更新 KV‑cache；  
  - 同时接收上一步 Performer 生成的干净潜在并解码为输出；  
  - 将新的 KV 分片发送至 Performer；  
  - Performer 利用完整 KV 上下文单独执行 flow‑matching 求解下一音频‑视频潜在单元。  
  这种调度使感知、状态更新、潜在去噪与解码重叠执行，将吞吐瓶颈压缩到 Performer 的生成时间 + 通信开销，只要该时间小于 160 ms 单元时长即可实时运行。结合 CUDA 图捕获、编译优化等，模型端信号到信号延迟约 200 ms。

- 该工作表明，实时多模态智能体应当从根本上设计成全双工系统，让视觉、语音和语言推理联合学习，而非事后拼装。

### 7. 优点
- **真正的原生化流式架构**：从感知到生成的每个部件都严格因果，使模型具备“立刻使用新观测”和“立刻发出生成结果”的能力。
- **单 Transformer 统一多模态交互**：将语音、文本、视频的输入 / 输出整合到一个因果流，消除了外部 ASR/TTS/动画模块及以文本为中介的表示瓶颈。
- **Thinker‑Performer 推理流水线**：在保持单一模型语义一致性的前提下，通过 KV‑cache 交换将最耗时的潜在生成分离到 Performer，并与感知、解码重叠，有效降低时延。
- **全双工自然交互**：联合训练使模型学会打断、重叠、主动发言等真实对话行为，并能在生成语音的同时保持视觉反馈，显著提升互动自然感。
- **低延迟指标明确**：给出了清晰的模型端及总交互延迟，并与现有系统按测量口径区分比较，透明性较高。
