## 摘要
服务于图像到视频生成的扩散模型计算开销高昂，给大规模部署带来了显著挑战。真实的图像到视频工作负载往往包含相似请求，例如重复的特效模板、相关主题以及反复出现的镜头布局。现有跨请求加速方法主要通过特征复用来利用这种冗余。我们观察到，相似的图像到视频请求还共享高度一致的稀疏注意力模式，使得历史稀疏掩码能够作为请求条件先验，几乎不产生在线掩码预测开销。我们提出了一个以稀疏性复用为核心的跨请求复用框架，并将特征复用作为可选扩展，受轻量级引导增强的保护。我们的稀疏性复用实现为共享稀疏掩码复用，即重用来自相似历史请求的高质量稀疏掩码，以避免逐请求的在线掩码预测。可选的特征复用对高度冗余的时空区域应用下采样计算，在保持效率增益的同时减轻边界伪影。引导增强在复用后强化图像/文本条件，缓解语义漂移和条件遵循问题。实验表明，默认的稀疏性复用配置在保持生成质量的同时，实现了2.16倍的加速。

## Abstract
Serving diffusion models for image-to-video generation is computationally expensive, posing significant challenges for large-scale deployment. Real I2V workloads often contain similar requests, such as repeated effect templates, related subjects, and recurring shot layouts. Existing cross-request acceleration methods mainly exploit this redundancy through feature reuse. We observe that similar I2V requests also share highly consistent sparse attention patterns, enabling historical sparse masks to serve as request-conditioned priors with almost no online mask-prediction overhead. We propose a cross-request reuse framework centered on \textbf{sparsity reuse}, with \textbf{feature reuse} as an optional extension safeguarded by a lightweight \textbf{guidance enhancement}. Our sparsity reuse is implemented as shared sparse mask reuse, which reuses high-quality sparse masks from similar historical requests to avoid per-request online mask prediction. Optional feature reuse applies downsampled computation to highly redundant spatiotemporal regions, mitigating boundary artifacts while preserving efficiency gains. Guidance enhancement reinforces image/text conditioning after reuse, mitigating semantic drift and condition-adherence issues. Experiments show that default sparsity reuse configuration preserves generation quality with a \textbf{2.16$\times$} speedup.

## 2. 论文提出的方法论

### 2.1 核心思想与框架概览
- 基于两个关键观察：
  1. 语义相似的I2V请求共享全局场景结构，其块级稀疏注意力模式高度相关，掩码可直接跨请求复用。
  2. 相同初始噪声的相似请求早期去噪轨迹高度重叠，使潜在特征复用成为可能。
- 默认路径：**稀疏性复用**（sparsity reuse）——检索并复用历史高质量稀疏掩码。
- 可选加速路径：**降采样潜在特征复用**（downsampled latent feature reuse）——在冗余时空区域进行下采样计算。
- 轻量级保护机制：**引导增强**（guidance enhancement）——弥补复用引起的图像/文本条件弱化。

### 2.2 稀疏性复用（Sparsity Reuse）
- **观察依据**：对于图像条件相似度（DINO）>0.4的请求，top‑p（p=0.9）稀疏掩码的IoU可达85%以上，覆盖率仅比同请求谕示低4.97个百分点，但远高于粗粒度在线预测（Sparge‑Attn）。
- **掩码缓存与检索**：
  - 为每个历史请求存储每步、每层、每头的块级稀疏掩码（通过稠密或高精度稀疏注意力获取）。
  - 使用文本提示和条件图像的嵌入构建向量数据库（如ChromaDB），新请求通过余弦相似度检索k最近邻；若相似度超过阈值τ，则直接复用其掩码；否则回退到在线预测或稠密注意力。
- **安全回退机制**（防止少量分布漂移导致信息丢失）：
  - **块对访问刷新**：在N个连续层（layer_flash=8）内维护已访问块对地图，最后一层强制激活未访问块对，确保每组层内所有Q/K块对至少被访问一次。
  - **最小top‑k保证**：掩码缓存时附加min_top_k约束，防止某些高度集中的注意力头仅保留极少数块对，降低跨请求漂移风险。
- **头级负载均衡**：由于每头稀疏度不同，在Ulysses序列并行中先按已知稀疏度重新排序头，贪心分配以均衡GPU负载，减少尾延迟（8 GPU下降低约7.28%端到端延迟）。
- **高效块稀疏内核**：fp16下使用FlashInfer，fp8下适配SpargeAttn的量化后端，跳过被剪枝块对，仅计算保留块对的QK^T、softmax和PV乘法。

### 2.3 降采样潜在特征复用（可选）
- **动机**：直接跳过高度相似的非计算区域会导致区域边界模糊和撕裂伪影；直接缓存非计算区域的KV状态则带来约160倍的存储开销。
- **方法**：对非计算区域应用**时空降采样池化**，然后将降采样后的非计算序列与活跃序列拼接，共同进入后续注意力层。这使活跃token获得几乎完整的全局感受野，以极小的计算和存储代价消除边界撕裂。

### 2.4 引导增强（Guidance Enhancement）
- **第一帧键增强**：在自注意力中对第一帧token的键乘以步依赖因子β（仅改变键，不改变查询和值），使后续帧更充分地读取第一帧信息，加强图像条件。
- **图像嵌入增强及衰减调度**：在块嵌入阶段，将条件图像的卷积分支放大α倍（相当于将特征图乘以α），β和α均在复用开始后启用并随去噪步数衰减，成本极低（<20 ms/步）。

## 6. 论文的主要结论与发现

- 相似I2V请求具有**高度一致的结构化稀疏注意力模式**，可跨请求复用，且无需在线掩码预测开销。
- 以**跨请求稀疏复用**为核心的Chorus II框架，默认配置（top_p=0.95, min_top_k=0.1）在4步蒸馏Wan2.2上**保持生成质量**，fp8后端**端到端加速2.16倍**，fp16后端1.31倍。
- 该框架在**速度‑质量帕累托前沿**上明显优于其他方法：与Sparge‑Attn相近延迟时，对稠密基线的保真度显著更高；达到SVG2级别保真度时，延迟约减半。
- 可选的特征复用路径可将加速比进一步提升至**2.59倍**，但保真度指标下降，更适合速度优先场景。
- 轻量的引导增强能够有效缓解复用带来的条件弱化和语义漂移，成本极低。
- 头级负载均衡进一步在8 GPU下降低约7.3%延迟。

## 8. 不足与局限

- **请求流依赖**：加速效果高度依赖请求间的语义相似度；若服务中相似请求占比低，回退频率升高，整体加速收益会显著下降。
